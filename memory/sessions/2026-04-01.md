## Problems Encountered

**Issue:** No attack paths discovered for the scanned environment

**Root Cause Analysis:**
- The attack path finder requires `exposes` edges from INTERNET to internal resources
- `exposes` edges are only created when:
  - Security group has ingress from `0.0.0.0/0` to EC2/RDS
  - S3 bucket has public access enabled
  - IAM role has `"Principal": "*"` trust policy
- The scanned environment had no such exposures (good security practice!)

**Visual Confirmation:**
- Internet node (red diamond) had no outgoing edges to VPC resources
- All visible edges were gray `connected_to` (topology only)
- No EC2/RDS/Lambda resources were present in the scanned subnets

**Conclusion:** Lack of attack paths is a **positive security finding** - no internet-exposed attack vectors exist in this environment.
